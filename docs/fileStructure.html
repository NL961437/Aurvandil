Source File Structure

Overview

The Aurvandil Source Files are organized similar to the Java Spring Web Framework and helps promote a more organized workspace. The purpose of each of the packages is described in more detail below.

Business Actions (BA)

Java files stored in the BA (Business Actions) package are designed to handle the main logic of the program. Business actions in this program make queries and additions to the PostgreSQL database and create HEALPix spheres.

Controllers

The controllers package is responsible for controllers that delegate actions to the program Business Actions (BA) package to complete the main functions of the program. In Aurvandil, the actions controlled are adding files, counting stars, creating and destroying HEALPix spheres, and running cone search queries.

Data Access Objects (DAO)

Data Access Objects handle the interactions between physical files and the information in the program. Instances of this include objects to read GAIA source files and write queries to CSV files.

Graphical User Interface Objects (GUI)

The Graphical User Interface (GUI) package handles the main Aurvandil GUI window that the user will see. Responsible for building the fields to be used for querying, database selection, and desired parameters.

Action Listeners

The Action Listeners package is found within the GUI package and is responsible for handling a variety of button actions in the GUI such as the "Run Query" and saving settings on window close.

Utilities (Util)

The Utilities package holds useful Java files with static methods that are routinely reused in different parts of the program. Some of these responsibilities are file management utilities and static methods to calculate and check values.

Main Aurvandil Controller

The main Aurvandil Controller is the Aurvandil.java file. The main controller handles the base operations of the program and is best used for debugging different parts of the program.
